實作指引

藍⾊的⾊票為 #36BBD9 ；⿊⾊的⾊票為 #000000 。

請優先使⽤ MaterialApp、Scaffold、ListView、Container、Column、Row、 Card...等元件 進⾏佈局編排。

請依循 DRY (Don't Repeat Yourself) 原則，盡可能減少重複。

請寫出易維護、易懂的程式碼，畢竟⼯程師需要互相合作，需要⼤家都看得懂程式 碼才⾏！

⾴⾯⼀：服務項⽬⾴⾯。

此⾴⾯的返回按鈕僅為⽰意，開發過程中請以此⾴為⾸⾴(該按鈕應該不會出現)。 此⾴⾯若有需要使⽤到 Icon 的地⽅可以參照 Material Icons。

品項約束條件：

i. 每個品項的初始數量皆為 0 。

ii. 最低數量為 0，最⾼數量為 10 。

iii. 按下下⼀步時， 只能有⼀個服務項⽬的數量⼤於 0 ，且 必須要有⼀個品項的數量⼤於 0 。

iv. 可以使⽤程式碼進⾏以上約束，或是跳出警⽰訊息來避免使⽤者進入下⼀個⾴ ⾯。切記以上條件如果沒有滿⾜就不能到下⼀⾴。

狀態管理：請使⽤ setState 更新⾴⾯的狀態。

⾴⾯跳轉：

i. 使⽤者按下下⼀步時，請跳轉到下⼀個⾴⾯。

ii. ⾴⾯跳轉時，請將 服務項⽬名稱、品項總數、總⾦額 帶入跳轉函式中，並顯⽰於 下⼀個⾴⾯。

⾴⾯⼆：結帳⾴⾯

請將附件中的 assets/icon.png 加入專案的依賴中。

此⾴⾯需要顯⽰上⼀個⾴⾯帶入的 服務項⽬名稱、品項總數、總⾦額 。 無論服務項⽬為何，⾴⾯中間都要顯⽰ assets/icon.png 。

結帳流程：

i. 按下結帳時，請跳出醒⽬的訊息 (如：Dialog、Toast) 來告知使⽤者結帳成功的 訊息。

ii. 若會跳出彈窗，請在使⽤者按下確認按鈕後回到上⼀個⾴⾯，並將上⼀個⾴⾯ 的狀態清除。

iii. 如果使⽤者在按下結帳之前回到上⼀⾴，上⼀⾴的狀態應該保留，不應該被清 除。
